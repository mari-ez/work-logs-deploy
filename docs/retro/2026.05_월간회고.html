월간회고 (김혜진) - 2026년 5월

2026-05 월간회고

이번 달 한 줄 요약

초반 플로우 공백을 메우며 에너빌드 디벨로퍼 3.5~5단계 기획을 완주하고, Case1 개발요청까지 도달했다.

Keep — 잘 됐고 계속할 것

  • 주간개발회의·전무님 리뷰 루프 유지 — 매주 검토→피드백→수정 사이클이 빠르게 돌아간 덕분에 방향이 크게 틀어지지 않았다.
  • 프로토타입 선행 — 5단계 시뮬레이션처럼 복잡한 화면도 프로토타입으로 먼저 구현해 논의를 구체화한 방식이 효과적이었다. 말로만 설명했다면 훨씬 오래 걸렸을 것.
  • 외부 접점 챙기기 — 이지스 자산운용 미팅, 하이멕 회신, 외주 디자이너 킥오프를 동시에 진행하면서도 내부 기획 흐름이 끊기지 않았다.
  • 단계 구조 확정 — 5단계(에너지 연계 시뮬레이션)→6단계(분석 보고서)의 단계 구조와 전체 flow 분기가 월 내에 확정된 것은 큰 성과다.

Problem — 어려웠거나 걸린 것

  • 플로우 설계 순서 역전 — 유저케이스별 전체 flow는 제품 설계 초반에 해뒀어야 할 작업인데, 5월에 들어서 진행하게 됐다. 뒤늦은 출발이 한 달 내내 부담으로 쌓였다.
  • 컨펌 프로세스의 물리적 시간 — 전무님이 개발 프로젝트 업무 흐름에 익숙하지 않아 소통 간극이 생겼고, 이를 맞추는 데만 상당한 시간이 소요됐다. 화면 페이지 단위, 항목 단위로 하나씩 검토받는 방식도 시간을 많이 잡아먹었다.
  • 5월 1주차 워킹데이 부족 — 대체공휴일로 실질 근무일이 적었는데, 해야 할 일의 밀도는 그대로였다. 초반 일정 버퍼가 없어 이후 주에 부담이 옮겨왔다.
  • alt 구성 미확정 — 5단계 alt(신재생에너지 설비구성 대안)가 박사님 쪽 논의를 기다리는 상태로 월을 넘겼다. 이 부분이 시뮬레이션·보고서 단계 요건 확정의 병목이 되고 있다.

Try — 다음 달 달리 해볼 것

  • 검토 단위를 묶어서 올리기 — 페이지 하나씩 올리는 대신, 플로우 단위로 묶어서 한 번에 리뷰 요청. 검토자가 흐름 안에서 판단할 수 있게 하면 왕복 횟수가 줄어들 것.
  • alt 구성 타임라인 명시적 합의 — 박사님과 확정 시점을 명시적으로 잡아두기. "기다리는 중"이 아니라 날짜를 박아두어야 6월 말 릴리즈 일정을 지킬 수 있다.
  • 6월 말 v.1 릴리즈 역산 계획 — 유저케이스별 화면 완성 + 개발요청 / alt 확정 + 시뮬레이션·보고서 요건 확정, 두 트랙을 역산해서 주차별 마감을 명시적으로 관리한다.

선생님 한마디

설계 순서가 뒤집혀 있었음에도 한 달 안에 3.5~5단계 기획을 완주하고 개발요청까지 도달한 건 분명 잘 해낸 것이다. 다만 "컨펌받는 데 시간이 많이 들었다"는 말이 계속 나온다는 건, 검토 방식 자체를 바꿀 타이밍이라는 신호다. 6월은 릴리즈 달이다. 지금까지 쌓아온 것들이 실제로 돌아가는 걸 볼 수 있는 달이기도 하다. 역산 계획표 하나 만들어두고 시작하면 훨씬 선명하게 움직일 수 있을 것.
