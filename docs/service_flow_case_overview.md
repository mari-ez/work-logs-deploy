## 1. 케이스 구분 개요

에너빌드 디벨로퍼는 사용자의 진입 방식과 대상 부지 성격에 따라 아래와 같이 케이스로 분기된다. 정비사업 케이스를 먼저, 정비사업 미해당(주소검색) 케이스를 이후에 배치한다.

| 케이스 | 진입 방식 | 건물 유형 | 정비사업 여부 | 웹 정보 유무 | 총 단계 수 | 화면설계 |
|---|---|---|---|---|---|---|
| Case 1 | 정비사업 검색 | 아파트(공동주택) 고정 | O | O (정비사업 관련 웹에서 수집 가능, 지역 무관) | 4단계 (간소화) | ✅ 완성 |
| Case 2 | 정비사업 검색 | 아파트(공동주택) 고정 | O | X (정보 없음) | 5단계 (B안 — 화면설계 진행 중) | 🕐 미착수 |
| Case 3 | 정비사업 검색 | 비아파트 건물 유형 (조사 진행 중) | O | TBD | TBD | 🕐 미착수 |
| Case 4 | 주소 검색 | — | 해당 주소지가 정비구역 | — | Case 1·2·3으로 전환 | 🕐 미착수 |
| Case 5-1 | 주소 검색 | 아파트(공동주택) 사용자 선택 | X | — | 5단계 | 🕐 미착수 |
| Case 5-2 | 주소 검색 | 일반건축물 사용자 선택 (사무실·상가·주상복합) | X | — | 6단계 | ✅ 완성 |
| Case 5-3 | 주소 검색 | 데이터센터 사용자 선택 | X | — | 6단계 | 🕐 미착수 |
| Case 6 | 정비사업 검색 또는 주소 검색 → **2단계에서 분기** | 대규모 또는 복합개발 | — | — | 에너지노 컨설팅 의뢰 안내 | 🕐 미착수 |

> **건물 유형 고정 원칙:** 정비사업 진입 시 건물 유형 아파트(공동주택)를 기본으로 한다. 단, 아파트 외 정비사업 케이스(Case 3) 분류 기준은 추가 조사 진행 중이며, 대규모·복합개발 프로젝트는 Case 6(컨설팅 플로우)으로 안내한다.

> **총 단계 수 안내:** 위 단계 수는 현재 시점의 기획 초안 기준이며, 상세 화면 기획 과정에서 각 케이스별로 1~2단계 내외로 조정될 수 있습니다.

> **Case 6 — 컨설팅 플로우:**
> - 다양한 용도의 건물을 복합적으로 배치하는 대규모·복합개발은 에너빌드 분석으로 커버하기 어려움
> - 복합개발 예시: 백현마이스 도시개발사업
> - **분기 시점 확정:** 2단계(건물 유형 선택 시점)에서 분기. (2026-05-26 결정) 1단계에서 대지 설정 후 총 면적 정보가 확보된 상태에서, Case 5-1/5-2/5-3와 동일한 시점(2단계)에 "대규모/복합개발" 선택지를 추가하는 방식. 대지면적 기반 자동 감지 여지도 생기며, 흐름 일관성이 유지됨. 자동 감지 조건은 Q5 판단 기준 확정 이후 정의.
> - 대규모·복합개발 판단 기준: 추후 정의 필요 → [[미결 Q5 참조]](#미결-항목-및-질문)

> **정비사업 구역면적 전제 정책 — 확정 (2026-06-08)**
> - 도정법에 따라 정비구역 지정 고시가 완료된 구역은 구역명칭·면적을 관보에 필수 명시해야 하므로, 에너빌드 서비스는 정비사업 케이스(Case 1·2·3·4)에서 구역면적값이 존재함을 전제로 한다.
> - 전산 반영 지연·변경 지정 처리 중·측량 불일치 등으로 온라인 조회가 일시적으로 불가한 경우는 서비스 범위 예외로 처리한다.
> - 구역면적 정보 확인 출처: 서울시는 서울시 정비사업 정비몽땅 및 서울 도시공간포털, 나머지 지역은 추가 조사 후 확정.

## 3. Case 2 — 정비사업 검색 + 웹 정보 없는 경우

### 🔄 플로우 재기획 (2026-06-08 B안 확정)

**논의에서 확정된 전제:**
- **에너빌드 제안값 = 법규 기반 산출값.** 별도 제안이 아니라 해당 정비구역 용도지역의 법정 한계값을 그대로 적용.
- 법규검토 값을 사용자에게 직접 표시한다 (Case 5 3단계 방식).
- 1단계(정비사업 검색)는 Case 1과 동일.

**논의 배경:**  
2026-06-05에 Case 1과 동일한 4단계로 단순화되었으나, 웹 정보가 없는 경우 실제 필요한 처리 흐름—법규검토 → 건축규모 산정 → AI 렌더링—이 반영되지 않아 재기획. 1단계 이후 단계에서 ①용적률/건폐율/층수/높이 법규 한계값 조회 ②법규 기반 건축규모 산정 ③AI 동배치 이미지 렌더링을 어떻게 구성할지 A안·B안 검토 중.

### 아파트 세대수 산정 정책 {#case2-세대수-산정}

대지면적·용적률·건폐율·높이제한으로 아파트 예상 층수 및 세대수를 산정한다. Case 5-1에도 동일하게 적용.

**층수 산정 공식:**

```
방법 A (높이 기반): 최대 층수 = 높이제한 ÷ 층고(3.0m)
방법 B (면적 기반): 최대 층수 = 지상 연면적 ÷ 건축면적
                              (건축면적 = 대지면적 × 건폐율)

→ 예상 층수 = min(A, B)
```

> 건폐율이 낮을수록 방법 B가 층수를 더 강하게 제약하는 경우가 많음.

**세대수 산정 공식:**

```
① 지상연면적 (용적률 산정 기준) = 대지면적 × 용적률
② 전용면적 합계 = 지상연면적 × 72%
③ 예상 총 세대수 = 전용면적 합계 ÷ 74㎡
   (평균 전용면적 74㎡ = 59㎡ × 40% + 84㎡ × 60%)
④ 59타입 세대수 = 예상 총 세대수 × 40%
⑤ 84타입 세대수 = 예상 총 세대수 × 60%
⑥ 교차 검증: 예상 층수(위 산정값) × 건축면적 ≥ 지상연면적
   → 불일치 시 세대수 하향 조정
```

**가정값 (기본값 고정):**

| 가정값 | 기본값 | 근거 |
|--------|--------|------|
| 전용률 | 72% | 공동주택 실무 전용률 범위 70-75% 중간값. 59㎡ 타입 약 70%, 84㎡ 타입 약 73-74% 수준이며, 두 타입 혼합 단지 기준으로 72% 채택 |
| 층고 | 3.0m | 업계 통용값 (법정 기준 없음). 일반 아파트 2.8-2.9m, 최근 신축·고급 단지 3.0-3.2m 수준. 3.0m는 검토 편의상 채택한 중간값 |
| 세대 구성비 | 59㎡ 40% / 84㎡ 60% | 서울 일반 분양 아파트 업계 통용 참고값 (특정 통계 데이터 출처 없음. 2026-06-08 기획 디폴트로 채택) |

> **홍교수님 검토 완료 (2026-06-10 개발주간회의 확인):** 전용률 기본값(72%), 층고 기본값(3.0m), 세대 구성비(59㎡ 40% / 84㎡ 60%), 세대수 산정 공식 전반 — 이대로 확정.

### 아파트 부대/복리시설 및 근린생활시설 면적 산정 정책 (2026-06-10 확정)

Case 2·5-1 아파트 규모 산정 단계에서 주차장 외 부대시설과 근린생활시설 면적을 아래 정책으로 산정한다. (홍교수님 통화 내용 반영)

#### 1. 규모 산정 결과 추가 표시 항목

아파트 규모 산정 결과에 다음 두 항목을 별도 표시한다.

| 항목 | 포함 범위 | 비고 |
|---|---|---|
| 부대/복리시설 | 주민공동시설 + 관리사무소 + 근로자휴게시설 | 주차장 제외, 근린생활시설 제외 |
| 근린생활시설 | 공동주택 단지 내 근린생활시설 | 건축물 용도: 제1종근린생활시설 |

주차장은 이미 별도 항목으로 산정하므로 부대/복리시설 면적에 포함하지 않는다.

#### 2. 부대/복리시설 면적 산정 정책

**법령 기준 (주택건설기준 등에 관한 규정):**

| 시설 | 조문 | 법령상 기준 |
|---|---|---|
| 주민공동시설 | 제55조의2 | 100~999세대: 세대수 × 2.5㎡ / 1,000세대↑: 500㎡ + 세대수 × 2㎡ |
| 관리사무소 + 근로자휴게시설 | 제28조 | 50세대↑: 10㎡ + (세대수 - 50) × 0.05㎡, 최대 100㎡ |

> 세대수별 의무 시설: 150세대↑ 경로당·어린이놀이터 / 300세대↑ + 어린이집 / 500세대↑ + 주민운동시설·작은도서관·다함께돌봄센터

**적용 정책 (대규모 단지 기준 단일 적용):**

초기 기획 검토 단계에서는 세대 구간별 분기 없이 1,000세대 이상 기준으로 단순화한다.

```
부대/복리시설 면적 = 주민공동시설 면적 + 관리사무소·근로자휴게시설 면적

주민공동시설 면적              = 500㎡ + 예상 세대수 × 2㎡
관리사무소·근로자휴게시설 면적 = min(100㎡, 10㎡ + (예상 세대수 - 50) × 0.05㎡)
```

예상 세대수는 [아파트 세대수 산정 정책](#case2-세대수-산정)을 그대로 사용한다.

**단순화 근거:**
- 1,000세대 전후 산정값 차이 미미: 999세대 기준 2,497.5㎡ vs 1,000세대 기준 2,500㎡
- 에너빌드 분석 대상 특성상 1,000세대 미만 아파트 단지 사례는 많지 않을 것으로 예상
- 초기 기획 단계 목적(대략적 규모·에너지 분석)에서 구간별 분기 불필요
- 100세대 미만(주민공동시설 의무 없음) / 50세대 미만(관리사무소 의무 없음) 소규모 단지는 향후 정확도 개선 단계에서 구간별 법정 산식 적용 재검토

#### 3. 근린생활시설 면적 산정 정책

건축물 용도: **제1종근린생활시설**
법령상 연면적 비율 상한 없음 (2014년 세대당 6㎡ 상한 폐지 — 주택건설기준 등에 관한 규정)

용도지역에 따라 아래 비율로 산정한다.

| 입지 조건 | 근린생활시설 면적 (전체 연면적 대비) |
|----------|-------------------------------|
| 준주거·상업지역 외 (일반) | **0.8%** |
| 준주거·상업지역 (역세권) | **10%** |

#### 4. 근린생활시설 추가 시 연동 산정

근린생활시설 면적이 추가되면 전체 연면적·용도별 면적·주차대수 산정이 연동된다.

```
총 주차대수 = 공동주택 주차대수 + 근린생활시설 주차대수
```

공동주택은 공동주택 기준(주택건설기준 제27조), 근린생활시설은 근린생활시설 기준(주차장법 시행령)으로 각각 산정 후 합산한다. 기존 주차대수 산정 정책을 그대로 따르는 것이며 신규 정책이 아니다.

#### 단계 구성

| 단계 | 내용 |
|---|---|
| **1단계** | 주소 검색 |
| **2단계** | 대지현황 확인: 대지위치(주소)·면적·지역지구 자동 제공 |
| **3단계** | 법규검토(개략): **데이터센터 건설 가능 여부 우선 표시** + 건폐율·용적률·층수·높이 제한 + **통상적인 인허가 기간 안내** |
| **3.5단계** | **IT 전력규모 입력:** 사용자가 선택 또는 직접 입력. 건축규모 산정의 기반이 되는 값. 초기 기획 단계이므로 약간의 오차 허용. |
| **4단계** | 규모검토(개략): IT 전력규모를 기반으로 건축면적·건폐율·연면적(지상)·용적률 제공 |
| **5단계** | 에너지 연계 시뮬레이션 (데이터센터 입지분석 요약 포함) |
| **6단계** | 에너빌드 분석 보고서 (데이터센터 입지분석 및 검토결과 상세 포함) |

> **IT 전력규모 입력 (3.5단계, 2026-06-05 추가):**  
> - 가장 처음 하는 검토는 땅의 주소로 데이터센터 건설 가능 여부 확인이다.  
> - IT 전력규모가 초반에 확정되어야 건축규모를 산정할 수 있다.  
> - 따라서 사용자가 IT 전력규모 값을 선택하거나 입력하는 단계가 필요하다.  
> - 초기 기획 단계의 규모산정이므로 약간의 오차는 허용한다.  
> - 규모 선택 옵션 및 직접 입력 방식은 기존 datacenter_scale_classification.md의 5단계 에너빌드 규모 구분(안) 참조.

> **통상적인 인허가 기간 안내 (3단계, 2026-06-05 추가):**  
> - 3단계에서 데이터센터 건설 가능 여부 표시와 함께 통상적인 인허가 기간을 안내한다.  
> - 구체적인 안내 문구 및 기간 데이터는 화면기획 시 별도 정의.

> **4단계 규모검토 — 지하/지상 층수 항목:** 지하/지상 층수는 서비스에서 계산하여 제공하기 어려운 항목으로, 제공 항목에서 제외. 지하층은 규모에 따라 상세 건물 설계 시 설정할 수 있다는 안내 문구를 제공한다. → [[미결 Q2 참조]](#미결-항목-및-질문)

> **5단계·6단계 입지분석 제공 방식:** 입지분석 내용은 5단계(에너지 연계 시뮬레이션)에서 간략 버전으로, 6단계(에너빌드 분석 보고서)에서 상세 버전으로 제공한다. 5단계와 6단계에서 각각 제공할 항목은 별도 정의 필요. (미결)
